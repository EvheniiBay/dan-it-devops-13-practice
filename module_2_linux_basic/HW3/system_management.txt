1.
Етапи завантаження Linux:
1. BIOS/UEFI та POST:
Після ввімкнення живлення запускається прошивка материнської плати (BIOS або UEFI).
Виконується POST (Power-On Self-Test) — перевірка справності обладнання (процесора, пам'яті тощо).
Прошивка шукає завантажувальний пристрій і передає керування коду завантажувача.
2. Завантажувач (Bootloader):
Найпопулярнішим є GRUB (або GRUB2).
Він пропонує меню вибору операційної системи або ядра.
Завантажувач копіює образ ядра (vmlinuz) та образ тимчасової файлової системи (initramfs) 
в оперативну пам'ять і запускає ядро.
3.Ініціалізація ядра (Kernel Initialization):
Ядро ініціалізує основне обладнання (драйвери, керування пам'яттю).
Монтується тимчасова файлова система initramfs, яка містить необхідні модулі 
для доступу до реального жорсткого диска.
Після цього ядро монтує основну кореневу файлову систему (/) і запускає перший процес у просторі користувача.
4. Процес Init (systemd):
Запускається перший процес із PID 1, який у сучасних дистрибутивах зазвичай є systemd (раніше init або upstart).
Він послідовно запускає системні служби (мережу, логування, графічний інтерфейс тощо) 
згідно з налаштованими цілями (targets або runlevels).
5. Вхід у систему (User Space):
Після запуску всіх служб з'являється вікно входу (Display Manager) або командний рядок.
Після автентифікації користувач отримує доступ до робочого оточення. 

2.
Для перегляду системних журналів в Ubuntu через термінал найчастіше використовують
команду journalctl або переглядають файли в директорії /var/log.
1. Використання journalctl (systemd logs)
Це основний інструмент у сучасній Ubuntu. 
- Перегляд усіх записів: journalctl (гортання стрілками, вихід — клавіша Q).
- Останні 20 рядків: journalctl -n 20.
- Режим реального часу: journalctl -f (показує нові записи в міру їх появи).
- Тільки помилки: journalctl -p err -b (помилки поточного завантаження).
- За певний час: journalctl --since "1 hour ago".
2. Перегляд текстових файлів у /var/log
Багато служб все ще записують дані у звичайні текстові файли. 
- Загальний системний журнал: cat /var/log/syslog (або less /var/log/syslog для зручного читання).
- Логи авторизації (sudo, вхід): cat /var/log/auth.log.
- Логи ядра (hardware): dmesg або cat /var/log/kern.log.
- Логи встановлення пакетів: cat /var/log/dpkg.log.

3.
Розбір дозволу -rw-------
Перший символ (-): Це звичайний файл (якби була d — це директорія).
Перша група (rw-): Права власника (Owner). Він може читати (r) та редагувати (w) файл, але не може його запускати.
Друга група (---): Права групи (Group). Учасники групи не мають жодних прав.
Третя група (---): Права всіх інших (Others). Повний доступ заборонено.
Отже тільки власник фійлу має право читани та редагувати його, але не виконувати його.
Числовий еквівалент: 600
Щоб додати прапорець виконання (x) для файлу використовується команда chmod. Є два основні способи:
1. Символьний метод
chmod u+x [назва_файлу]
2. Числовий метод
Щоб змінити -rw------- на -rwx------, сума цифр для власника зміниться з 6 (4+2) на 7 (4+2+1).
chmod 700 [назва_файлу]

4.
- dpkg (Debian Package)
Це низькорівнева утиліта. 
Вона вміє працювати тільки з конкретним .deb файлом, який вже є на диску.
dpkg встановлює, видаляє або розпаковує окремий файл, але не вміє завантажувати файли з інтернету і не вирішує залежності. 
Якщо пакету потрібна бібліотека, якої немає, dpkg просто видасть помилку і зупиниться.
- apt (Advanced Package Tool)
Це високорівнева надбудова над dpkg.
apt шукає пакети в онлайн-репозиторіях, завантажує їх та автоматично встановлює всі необхідні залежності.
| Критерій       | apt           | dpkg                   |
| -------------- | ------------- | ---------------------- |
| Рівень         | Високий       | Низький                |
| Репозиторії    |+              | -                      |
| Залежності     |+              | -                      |
| Формат         |.deb           |.deb                    |
| Рекомендований |+              |+/-                     |
